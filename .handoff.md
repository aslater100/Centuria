## 4. GOTCHAS / DISCIPLINE (save the next agent time)

- **Economy byte-identity is the Phase-A/B contract.** `buildings[]` + `buildingBonus`
  are untouched; `placedBuildings` is additive. region-longrun/economy-balance stay
  green. Phase C deliberately breaks this — re-baseline, don't expect byte-identity.
- **Determinism:** new sim work is free-function-style, RNG-order-preserving. The
  `launchExpedition`→`launchExpeditionTo` refactor kept the name-draw RNG in place;
  `autoPlaceCell`/`ensurePlacements` use **no RNG**. region-longrun is the guard.
- **Saves:** every new Settlement/sim field must serialize (the top serialize spreads
  `...s` for settlements) + backfill in `deserialize` + be added to the ~4 settlement
  creation literals (tsc catches misses). `placedBuildings` migrates old saves on load.
- **AI is the WRONG tool for crisp foreground sprites/UI/town icons** (proven this
  session — generated town sprites looked bad at ~50px). The win is **procedural
  rendering + the spatial layer**, not text-to-image. The `gen:local` tool + the
  `asset-generator` agent exist for backdrops/atmosphere only.
- **`gen:local` (local AI image tool)** runs against a LOCAL SD server (A1111/ComfyUI

## 5. UI HOOKS (for the spatial work)

- Map click handler (`regionview.ts`): mode branches in priority order — building
  placement → founding → scout → settlement-select → claim-land. Mirror these for new
  modes. `screenToHex(mx,my,size,ox,oy)` maps clicks to hexes; cell key = `col*REGION_N+row`.
- Per-frame overlays (`drawFoundingOverlay`/`drawBuildingPlacementOverlay`/
  `drawPlacedBuildings`) draw in map-space under the camera transform, after the
  mapCache blit. Helpers: `box/roof/litWindow/shade/smoke` (Phase-1), `fillHexPath/
  strokeHexPath`, `hexCenter/hexCorners/hexLayoutParams`.
- Territory/borders are baked into the **static mapCache** (`drawTerritories`),
  rebuilt on signature change — dynamic overlays must be per-frame, not in the cache.
