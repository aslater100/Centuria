## 2. COMPLETED (this session — PR-3 slice 3: per-good local prices + macro coupling)

User said "Keep working" → the economy track's headline next step was **PR-3 slice 3**
(per-good local prices, "the first slice that moves the macro economy"). Every prerequisite
was in place (slice 2's per-town gate, slice 1's cargo, the C1 extractions), so this session
built the full slice in `systems/goods.ts` + `systems/arbitrage.ts` + the macro seam in
`region.ts`.

- **Per-good local PRICES (`localGoodPrice`, systems/goods.ts).** Each tracked good is now
  priced PER TOWN: `basePrice × (1 + scarcity × GAIN)`, scarcity = `clamp(1 − stock/demand)`.
  `localGoodDemand` = the town's full-supply consumption appetite for the good (Σ over the
  goods it makes that consume it). A producer flush with a good prices it at base; a town
  that consumes it but holds none prices it dear. `basePrice` = 1 + #intermediate inputs
  (refinement). Pure reads off `goodStocks` — **no raw is ever priced** (the load-bearing
  "raws never in goodStocks" invariant the slice-2 review flagged).
- **Price-driven ARBITRAGE (systems/arbitrage.ts).** Dropped the wage-gap proxy: each lane
  now ships the good with the largest profitable **price gap**, from the cheap (abundant)
  town to the dear (short) one — a deprived town pulls exactly the good it needs (where the
  proxy shipped `goodIds[0]` regardless). `pendingIncome = cargo × gap × (1 − tariff) × 5`.
- **The macro coupling (region.ts).** A nation-wide **`localGoodsScarcity`** index ∈ [0,1],
  cached each month from the per-town production **GATES** (lost output / potential output),
  now feeds: (a) cost-push inflation `+= scarcity × LOCAL_GOODS_INFLATION` (0.08), and
  (b) an industry-output drag `× (1 − scarcity × LOCAL_GOODS_OUTPUT_DRAG)` (0.10), both one
  month lagged like `supplyShockMult`. New serialized field `localGoodsScarcity` (backfill 0).
- **⚠️ THE KEY FINDING — the coupling is INERT in current balanced play (markers byte-identical
  to base), which CORRECTS the prior baton's "the markers WILL move from base."** The index is
  driven off the gate, so it's **exactly 0 whenever every gate is 1** — and real autoplay towns
  are **MIXED** (both industry & agriculture output), hence self-sufficient, hence never strand
  a cross-sector good. Measured: `localGoodsScarcity` is 0 in **all** 8 headless seeds across
  181y (a clean probe, before wiring). So slice 3 ships the full machinery with **ZERO balance
  risk today**; it bites only under **SPECIALISATION** (forced fixtures prove it: a pure-industry
  + pure-agri 2-town nation gives index ≈ 0.14, textiles priced 2.0 in the industry town vs 1.0
  in the agri town → arbitrage ships textiles to the industry town). **Phase C (spatial terrain
  yields) is what will make towns specialise and turn this coupling on** — slice 3 + Phase C
  together produce the gameplay. Arbitrage is likewise dormant in autoplay (no price gaps →
  no shipment; also no routed multi-town player pairs in the headless seeds).
- **Why the gate (not stock magnitudes) drives the macro index.** A stock-based index would
  also fire during a RAW shock (level<1 depletes stocks) → double-counting the raw cascade AND
  breaking single-town byte-identity-during-shock. The gate is the **pure** slice-2-divergence
  signal: for a self-sufficient town `have/need ≥ 1` regardless of `level` (both production and
  consumption scale with level), so the gate stays 1 in boom OR shock → index 0 → no
  double-count. (The supply-cost-push single-town oil-embargo tests still pass unchanged.)
- **Gates.** tsc clean; **1003 tests** (988 + 15 new in `tests/goods-prices.test.ts`: demand,
  prices, the index 0/positive/bounded/relieved-by-shipping, a cost-push test mirroring

## 4. FAILED PATHS / GOTCHAS (save the next agent time)

- **The slice-3 macro coupling does NOT move today's economy — and that's correct, not a bug.**
  It's gate-driven, so it needs SPECIALISED towns to bite; autoplay towns are mixed/self-sufficient
  → index 0 → byte-identical markers. Don't "fix" this by switching the index to stock magnitudes
  (that double-counts the raw cascade and breaks single-town byte-identity-during-shock — see §2).
  The right activator is **town specialisation** (Phase C terrain yields, or a future demand/
  specialisation mechanic). Verify the coupling FIRES with a forced pure-industry/pure-agri fixture
  (`tests/goods-prices.test.ts`), not with the headless sweep.
- **Two price notions, deliberately distinct.** The ARBITRAGE price (`localGoodPrice`) is
  **stock/demand** based (so a town low on stock pulls a shipment). The MACRO index
  (`localGoodsScarcity`) is **gate** based (lost output / potential). They agree under
  specialisation; they differ under a raw shock (gate stays 0 for self-sufficient towns, stock
  would not) — that difference is intentional (no double-count).
- **`localGoodsScarcity` is a THIRD one-month-lagged cache** (after `supplyShockMult` and
  `_electronicsDisrupted`): set at the END of `tickIntermediateGoods` (no RNG), read NEXT month by
  `tickMonetary` (cost-push) + `updateSectors` (industry drag). Serialized with backfill 0.
- **`monthlyUpdate()` runs every 30 days (`day % 30 === 0`), NOT every `r.month`** (which uses
  `DAYS_PER_MONTH = 5`). `updateSectors` is early in it, `tickIntermediateGoods` late — so a value
  injected before a 30-day boundary is read by `updateSectors` then recomputed by
  `tickIntermediateGoods` in the same block (the trick the output-drag test uses).
- **C1 extraction recipe (proven 5×) + the systems/region.ts import cycle** are unchanged from the
  prior baton — `systems/arbitrage.ts` AND `systems/goods.ts` import values from `region.ts`
  call-time-only; keep new `systems/` value-imports call-time-only. `tradeFlows[].cargo` is the
  physical-units field; dispatch debits the SOURCE (now the cheaper-priced town), arrival credits
  the destination.
- **`Settlement.goodStocks` is THE store**; go through the accessors. To seed a town directly in a
  test, poke `(s.goodStocks ??= {})['id']`.
- **`auxRng` is a THIRD serialized RNG** (notable health, loan ids). New incidental stochastic
  detail belongs on `auxRng`.
- **The two gates for any `region.ts` economy change:** the determinism harness
  (`tests/serialize-determinism.test.ts`) and the save-size guard (`tests/save-size.test.ts`).
  For a "macro-neutral / byte-identical" claim, ALSO stash-and-diff the headless sweep.
